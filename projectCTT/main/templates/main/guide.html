Objective

The main objective of this system is to support the contact tracing of patients of infectious diseases (e.g., COVID-19, MERS etc.). The contact tracing essentially finds users who have come in direct contact with an infected user (e.g., Covid-19 patient) or who are in contact with other users who have already been in contact earlier with the patient.


Reference

  • This demonstration is based on our research on efficient indexing techniques for the contact tracing query. The technical report can be found at arXiv.
  • A demonstration paper has been recently accepted in ACM SIGSPATIAL 2020.


Dataset

We have used mobility trace data of mobile users of the largest telecommunication operator in Bangladesh. Thus the database we used in this demonstration contains spatio-temporal traces (location, time-stamp) of all users collected by the telecommunication company through mobile GPRS. All the mobile numbers are anonymized using a distinct anonymous ID. Henceforth, anonymous ID and user ID will mean the same thing.

Technical Details

The contact tracing of a virus infected person (e.g., a Covid-19 patient) is challenging as they encounter lots of other users in their journey for the last 14 days. We assume that we have access to the mobility traces of all users in the system. Now, when a new Covid-19 patient is identified, we first search for his (or her) mobility traces in the database using his/her id (e.g., mobile phone number). After finding his/her mobility trace as a sequence of (location, time) pairs, we need to find who are other users whose (location, time) pair matches with any of the (location, time) pairs of the Covid-19 patient. Having a database of size in terabytes containing the mobility traces of millions of users for the last 14 days, it is a daunting task to find such a match. A traditional approach using existing spatio-temporal indexes, e.g., 3DR-tree, to solve this problem requires hours to find the contact trace of a single patient. We develop a hierarchical multi-level index, namely QzR-tree that can efficiently find matches between users and the virus infected patient. Please see the technical report for details.

Privacy Policy

The assumption is that we have a database of all users' mobility data collected by the telecommunication companies. All the mobile numbers are anonymized using a distinct anonymous ID.

The system has broadly two authorization levels for now:

  • Tracers: These users have the access to trace and analyze mobility/contact given anonymous IDs. These users are never shown any identification information (e.g., mobile number) about the users.
  • Code Breaker: These users have the access to code-decode between mobile numbers and their anonymous IDs. Each such coding/decoding query is logged and subject to review by the central authority. These users also DO NOT have access to the query interfaces and MUST not have any access to a separate tracer account.

Notes

  • The radius of a circle at any location proportionally increases with the cumulative time spent at that location.
  • On the other hand, opacity of a circle is determined by the number of visits to that location. More visits to a location will make the circle more opaque.
